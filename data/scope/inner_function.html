IIFE Pattern

  • Global variables are considered bad practice, they can potentially be modified from anywhere and any part of the program may depend on it. These dependencies increases complexity in software system.
  • We can take advantage of the javascript function scope property to encapsulate global variables and making them private to that function. Note that function call is necessary to assure the code run
  • A more compact way to do the same think can be done using the Immediately Invoked function Expressions (IIFEs) Pattern. TO REMOVE ?It is based on 1) An anonymouse function is used and 2) the Immediately-invoked function expressions
  • Javascript does not support block level scoping. IIFE Pattern can be used to simulate them. (Note how variable can be passed to the IIFE Scope)


    // Module A-
    user = {...}
    foo  = function (){....}  //global

            

    // Module B
    foo  = function (){....}  //global

            

    function run () {
        a = 10;                  // private
        foo  = function (){...}  // private
        foo (5);
    };
    run ();

            

    (function() {
        a = 10;                  // private
        foo  = function (){...}  // private
        foo (5);
    })();
